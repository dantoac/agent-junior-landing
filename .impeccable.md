# AgenticJunior - Design Context

## Design Context

### Users
- **Primary audience**: Broad -- from solopreneurs and small teams (2-10) to tech-savvy professionals (devs, PMs, consultants) AND everyday users who are NOT tech-savvy. The interface must be approachable for non-technical people while still feeling capable for power users.
- **Context**: People who want AI-powered productivity (email, calendar, tasks, GitHub) without the burden of self-hosting or managing infrastructure
- **Job to be done**: Automate routine work (triage email, schedule meetings, manage tasks) so they can focus on high-value work
- **Emotional goal**: Feel confident that their data is secure and the tool is reliable -- not impressed by flashy AI demos, but reassured by solid execution

### Brand Personality
- **Three words**: Confiable, Limpio, Profesional
- **Voice**: Direct and clear. No hype, no buzzwords. Explain what it does, prove it works.
- **Tone**: Like a competent colleague -- professional but not corporate, helpful but not chatty

### Aesthetic Direction
- **Visual tone**: Editorial modern -- clean typography, generous whitespace, purposeful color accents
- **References**: Stripe, Resend -- professional gradients (subtle, not neon), strong typography hierarchy, data-forward layouts, light editorial feel
- **Anti-references**: ChatGPT / generic AI aesthetic -- no cyan-on-dark glow, no chat bubble UIs as hero, no "AI slop" patterns (glassmorphism everywhere, sparklines as decoration, gradient text on every heading)
- **Theme**: Both light and dark, dark as default. Blue primary (hue 250) with cyan secondary (hue 195)
- **Typography**: DM Sans (display/headings) + Source Sans 3 (body) -- already established
- **Color system**: OKLCH-based with DaisyUI v5 tokens. Blue primary conveys trust; cyan secondary adds energy without being gimmicky

### Design Principles
1. **Substance over spectacle**: Every visual element must serve a purpose. No decoration for decoration's sake. If a gradient, animation, or card doesn't help the user understand or act, remove it.
2. **Trust through restraint**: Security and reliability are core differentiators. The design should feel solid and considered, not experimental. Fewer elements, done well.
3. **Progressive disclosure**: Start simple, reveal capability through interaction. Don't overwhelm with features on first view -- let users discover depth.
4. **Respect the user's time**: Fast loads, clear hierarchy, scannable content. No walls of text, no redundant sections. Every word earns its place.
5. **Platform-agnostic quality**: Must look polished on mobile, tablet, and desktop. Not just responsive -- adapted. Different layouts, not just scaled-down versions.

### Technical Stack (Landing)
- **Framework**: Vanilla HTML + CSS + JS (no build step)
- **CSS**: Tailwind CSS v4 via CDN + DaisyUI v5
- **Fonts**: Google Fonts (DM Sans + Source Sans 3)
- **i18n**: Custom JS-based (EN/ES)
- **Hosting**: GitHub Pages
